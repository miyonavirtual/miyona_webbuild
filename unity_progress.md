## 5. Development Constraints & Challenges (Logs)
- **Folder Disconnection Constraint:** The AI Agent is operating inside the Next.js web project repository (`/miyona.ai`). The actual Unity Editor project is completely disconnected from this folder path. Consequently, scripts generated by the AI must be manually copy-pasted or recreated inside the Unity Editor by the user; they cannot be automatically injected into the Unity hierarchy.
- **VRM Version Conflicts (Resolved):** The project relies on the modern **UniVRM 1.0** package. Initial facial animation and tracking scripts were written using the older VRM 0.x API (e.g., `VRMBlendShapeProxy`, `VRMLookAtHead`), which resulted in `namespace could not be found` compile errors. The scripts (`MiyonaEmotionController.cs`, `MiyonaEyeContact.cs`) were successfully rewritten and migrated to use the `UniVRM10` API (e.g., `Vrm10Instance`, `Runtime.Expression`, `ExpressionPreset`).
- **Input System Conflicts (Resolved):** The Unity project is configured to use the new Unity Input System package. Legacy input code (`Input.GetAxis()`) threw `InvalidOperationException` errors, necessitating updates to use `UnityEngine.InputSystem` (e.g., `Mouse.current`).
